CRITICAL CORRECTION — injection formula
EARLIER 2C - I WAS WRONG (sign error). Correct injection:
map_injected = map_observed + I_bs      (masked to footprint)
Derivation: at a site whose observed bg matches the model cloud level (matched
within sqrt2), additive = (sky with core) - (sky without core) = raw - const = I.
The crater is included automatically (obs+I ~ raw = core + cratered cloud).
Equivalent "dig crater (C-I) + add core C" reduces to +I. ONLY bs is needed; C
cancels. getsf then subtracts its own rim background -> recovers ~bs flux =
M_SED3bs (biased) -> correction -> M_BE. So getsf's own bg step produces the
over-subtraction, exactly as on real data. (Fixed in draft 3.8, inject.py,
injection_test_notes.md.)

FLATTENING (controlled background) — decision and method

PRIMARY accuracy test: inject onto a flattened background (remove sub-source
cloud fluctuations) so the test measures the SYSTEMATIC over-subtraction, not
random cloud-fluctuation scatter. LEGITIMATE because the crater/over-subtraction
bias survives flattening; only random scatter is removed.
SECONDARY realism test: inject onto RAW cloud (flatten=False) to characterize
the cloud-fluctuation scatter (real-world error budget; correction passes it
~1:1, can't remove it).
METHOD: biharmonic inpainting of the footprint from the just-outside rim
(getsf-style core/bg separation). NOT a plane (plane over-flattens bright
structured regions -> artifact disk). skimage.restoration.inpaint_biharmonic.

CONTRAST / DETECTABILITY — gate for which models to inject

Completeness criterion (user): peak SURFACE-DENSITY excess >= 0.5 * background
= total contrast (bg+excess)/bg >= 1.5.
REAL Aquila detection limit is lower: smallest detected SD contrast ~1.1
(excess/bg ~0.1); per-band INTENSITY contrasts run near or below 1 for some
cores (over-subtraction signature). Most real SD contrasts < 1.5, few > 3.
Use gate contrast >= 1.1 (excess/bg >= 0.1) to mirror real detections: admits
270/293 production + 31/36 subdivision models; faint cores carry the LARGEST
bias (up to 1.6-1.9), exactly the science target.
Intensity contrast (0.6% for i4j2k5 at 3.39e22) was misleading; SURFACE-DENSITY
contrast is the right currency (detection is on the Sigma image — see below).

KEY PHYSICS POINT — detection is on the Sigma image
User's benchmarking: detection on the hires Sigma image alone gives ~best
completeness (cores clearer there than in any single band). So the Sigma image
is the PRIMARY detection channel; bands feed the SED/flux measurement.
DISTRIBUTION MATCHING — design boundary (DECISION)

Matching the injected population's (T_SED, SD-contrast, M) distribution to the
real Aquila detected population is needed ONLY for completeness estimation /
assessing CMF parameter quality on REAL extractions (Paper II).
For validating the METHOD itself (this paper), NO matching needed: just span
the parameter space with detectable models (contrast >= 1.1, low-contrast

The three-population comparison (magenta input -> green recovered -> grey

FINDINGS:

The hires column is ~13-14% BELOW the model EVERYWHERE, incl the cloud baseline
(flat outer reldiff ~-14% at all resolutions; extreme corners ~-72% edge
artifacts). RESOLVED: this is a TEMPERATURE bias, NOT a normalization artifact.
Ruled OUT by checks: opacity (O&H col4 is an EXACT beta=2 power-law,
kappa=83.364*(100/lambda)^2, identical to the fit power-law at 160/250/350/500
when both =10 at 300; normalization x1.0796); mu (2.8 in both fitfluxes and
radmc3d); dust-to-gas (0.01 both, consistent); optical depth (tau_300~0.0035 in
radmc3d -> optically thin, tau_160~0.012, <1% effect). By elimination -> TEMP:
the cloud is NOT isothermal. Uniform-density medium externally heated by ISRF;
at N=3e21 the LOS carries A_V~3 mag -> self-shielding -> warm illuminated
surface, cooler shielded interior -> LOS T gradient. Single-T MBB fit (160-500)
returns luminosity-weighted (warm-biased) T -> N ∝ 1/B(T) comes out ~14% low,
uniform because the cloud is uniform. Core stacks its steeper gradient on top.
=> the -14% is the genuine temperature systematic the method corrects (shows up
in cloud too), NOT an artifact to divide out; injection->getsf->correction
carries it end to end. PENDING confirmation: radmc3d cloud dust-T structure
(expect gradient) + T_fit vs mass-weighted T (expect fit ~1-2 K warmer); then
can predict -14% exactly by running single-T fit on the cloud LOS SED.
CONFIRMED: user reports cloud T = 20 K outer edge -> ~13 K a factor-2 in radius
deeper -> isothermal 13 K -> drops further inside core. Single-T fit (160-500,
optically thin) on this profile reproduces -14% (Sigma_fit/Sigma_true=0.857,
T_fit~16 K) for full-gradient sightlines. Uniform across the map because the
73x73 box sits INSIDE the cloud (every sightline carries the full gradient; no
spherical edge in frame). Mechanism: warm outer dust dominates emission (B
steep with T, esp 160um), fit assigns that warm T, cold interior (most mass,
little emission) under-counted -> underestimate. Fully accounted for; physics
not calibration; carried end-to-end by injection->getsf->correction.

TWO getsf MASS TYPES — to compare (user's 2016 paper)

SED total-flux mass (single-T fit of source's integrated fluxes): the
M_SED-type bias; factors 2-5 for extreme temperature-excess cores.
Surfdens-integration mass (integrate hires per-pixel column in footprint):
more accurate for RESOLVED objects (Men'shchikov 2016: intensity fitting beats
flux fitting for resolved objects; best with 2-3 longest wavelengths; bg
subtraction a likely source of large systematic errors; unresolved masses
uncertain factor 2-3). PLAN: pull BOTH from getsf in the recovery test, compare
to truth, likely prefer surfdens for resolved / SED for unresolved; build/
validate the RT correction on whichever (or both). Measure empirically (with
reconstructed hires resolution the per-pixel gain mixes with spillover).
