Working summary 2 — RT-based core-mass correction method
Compiled from conversation, June 2026. Complements "Summary from previous chat."
 
1. The method
Per-core radiative-transfer (RT) correction of Herschel SED-derived prestellar core masses for Aquila. Rather than using RT models to populate backgrounds for completeness (the older framing in Summary 1), the method computes an RT model per extracted core and fits it to the observed wavelength-dependent contrasts and sizes, recovering accurate mass and temperature and correcting the isothermal modified-blackbody (MBB) SED-fitting bias. Completeness is decoupled from mass accuracy and handled separately (corrected RT models scaled down in mass, injected at small position offsets, re-run through getsf; plus a cheap SNR-based "theoretical completeness" precursor read directly off the grid).
Observables: signed center/peak intensity (contrast) and FWHM at each of the five Herschel bands (70, 160, 250, 350, 500 μm), up to ten observables per core. Band-dependent FWHM is the temperature-gradient diagnostic. Signed center intensity (not peak) handles donut/absorption morphology — negative values at 70 μm for cold cores in absorption are physically real.
 
2. Model family and parameters
Critical Bonnor-Ebert spheres. Parsimonious: the critical BE profile is a unique self-similar shape in log-log regardless of T_BE (T_BE is only a scaling parameter — it does not change the profile shape). Non-spherical geometries rejected: too many unconstrained parameters, and convolution washes out shape differences at Herschel resolution (~5% effect, below noise); defensible in the paper via the resolution argument, so no geometric-sensitivity test needed. Free parameters: M and R (equivalently T_BE and ρ_c, bijective for critical BE). G0 fixed (3D embedding makes per-core G0 unrecoverable). Protostellar cores add L_int. Fitting via Levenberg-Marquardt (scipy.optimize.least_squares), residual vector over observables normalized by σ (~5–10% calibration plus per-core/per-band background systematic), initialized from SED-derived values, evaluated against a pre-computed interpolated grid.
 
3. Grid and RT code
Switched from the old ray-tracing code modust to RADMC-3D (Monte Carlo). modust showed ~10–20% nonmonotonic intensity oscillations in the densest models (numerical artifacts) and under-heats the interior by ~3 K (a converged physics/implementation limitation in diffuse re-emission, not a convergence issue — classical and approximate lambda iteration give the same result). RADMC-3D is trusted.
Grid: 9 T_BE values (factor-2 spacing, 0.875–224 K — the wide range only spans M–R and is fine since T_BE is a scaling parameter), ~15 ρ_c per population (factor 1.44), ~100 radial zones (800 was over-resolving the flat BE center and hurting photon statistics; ~100 gives better statistics), MRW enabled. Sub-100 AU structure needs no special treatment because the ~100 AU surface-density pixel plus beam convolution plus 3″ resampling already wash it out. Store per-band PSF-convolved postage-stamp images (not scalars — contrasts only emerge after embedding on the background). Per-core fit cost drops from ~15 min (RT-in-loop) to seconds; verify interpolation with hold-out (<5% error) and a dedicated RT run at best-fit. Embedding cloud is a uniform-density sphere, R_cloud = 1.4e5 AU (≫ R_BE, fixed — the beam never reaches the boundary), with crater (limb-brightened) background implemented alongside traditional interpolated subtraction. Cloud column N_cloud added as a third grid dimension (flat appended table; row order irrelevant for interpolation), set per core from the observed local background; G0 fixed and the RT computes cloud temperature self-consistently.
 
4. Calibration arc (resolved)
PAH bands: modust's ISRF had six strong (~30×) PAH bands (3–12 μm); adding them to RADMC-3D changed temperatures by only ~1–2% (diffuse re-emission dominates core-center heating). Decision: omit PAH — unconstrained, non-universal, negligible in correct RT. modust's apparent PAH dependence was compensating for its interior under-heating.
Code cross-validation: in matched configurations the two codes agree (cloud ~12 K for the test model; modust+PAH ≈ RADMC-3D, modust−PAH ~3 K colder). An early alarming 6-vs-12 K core-center gap was traced to a wrong number from a different massive model and corrected.
G0 normalization: the ISRF is tabulated Black plus a scaled hot blackbody (peak 0.2–0.3 μm), implemented in both codes. The "1.22 vs 1.6" discrepancy was a missing-4π miscommunication, not a normalization error: the reported FUV integral 1.22e-3 was without the 4π; with it, G0 = 4π × 1.22e-3 / 1.6e-3 ≈ 10. The baseline field was always correctly G0 ≈ 10. A minor (~2%) trapezoid bug integrating just below the Habing band edge (912–2070 Å) was fixed. Report G0 ≈ 10 in the methods section.
Temperature calibration (key): calibrate to observed Aquila temperatures, not code-matching. Cores live in filamentary environments with T_d ~10–15 K (avg 13 K), Σ ~7e21–7e22 (avg 2e22, A_V ~10), not the diffuse cloud. At G0 = 10, Σ = 2e22: cloud surface 16.3 K, center next to core 8.9 K, and the hires SED-fit cloud temperature is 13.5 K ≈ observed 13 K. This confirms G0 = 10. Earlier pushes toward G0 = 30/60 came from comparing model central T to observed SED T — a quantity mismatch; SED T must be compared to SED T. Cloud-temperature variation across the observed range comes from N_cloud (column), not G0; the shielded cloud interior is nearly G0-insensitive at high column, while only the SED-weighted surface is G0-sensitive.
 
5. Validation result — crater vs interpolated background
Current, clean result (continuous 0.1 M⊙ core, 2e22 cloud, G0 = 10):
 
Crater (limb-brightened) background: T_SED = 8.52 K, M = 0.1015 M⊙ (≈ true 0.1) — accurate in both.
Interpolated background: T_SED = 8.26 K (cold-biased), M = 0.0676 M⊙ (~1.5× low).
350 μm peak flux a factor ~2 lower for interpolated, because it subtracts the edge cloud value across the footprint while the actual cloud under the core (the crater) is lower — over-subtraction.
Arithmetic closes: M ∝ F/B(T); flux ~0.5 × B_ν(8.52)/B_ν(8.26) ≈ 1.16 at 350 μm gives ~0.58, vs. observed 0.0676/0.1015 = 0.67 (approximate, since mass comes from the full multi-band SED fit, not the 350 μm peak alone).
 
SUPERSEDED — do not reuse: the earlier 0.05 M⊙ numbers (interpolated 0.0178, crater 0.0507, mass factor ~2.8, 350 μm peak factor ~5) were contaminated by a modeling artifact — that core's outer-radius density dipped below the cloud density, creating a spurious deficit and a genuine negative depression in the images. The factor-5 claim (and the speculation that it directly explains the original factor-5 surface-density discrepancy) is withdrawn. The clean effect at this mass/column is factor ~2 in peak, ~1.5 in mass; it may grow at higher columns and lower masses, to be mapped rather than asserted.
Two mass-under-estimation mechanisms, both corrected by the method: (a) temperature-gradient warm bias (high-mass cores → SED over-estimates T → under-estimates mass); (b) interpolated-background over-subtraction → flux loss (low-mass cores → under-estimates mass). Different physics, same direction; the crater background plus RT temperature structure corrects both because the forward model reproduces the full chain (emission → displacement → background subtraction).
 
6. Grid-construction constraint
Every embedded model must have the core join the cloud continuously: ρ_BE(boundary) ≥ ρ_cloud. When the BE outer density falls below the cloud density the model is a hole, not a core, and produces spurious depressions (this caused the 0.05 M⊙ artifact). This defines a natural minimum core mass at each N_cloud (the model where ρ_boundary = ρ_cloud). Enforce it as the low-mass edge of the grid per column, and check existing grid models for violations.
